    TPOMS

    TP OMS Configuration

    This section defines the supported Third-Party OMS (Order Management Systems) along with their authentication behavior and required input parameters.

    🟢 Kite (Zerodha)

    OMS Name: Kite
    Self Login: ✅ Yes

    Login Configuration

    • Login URL: https://kite.zerodha.com/connect/login
    • Token URL: https://api.kite.trade/session/token

    Required Inputs

    Field Name Label Type Required Notes
    UserId User ID Text ✅ Minimum 3 characters
    Password Password Password ✅
    ApiKey API Key Text ✅
    ApiSecret API Secret Password ✅
    TotpSecret TOTP Secret Password ✅
    PrimaryIpAddress Primary IP Address IP Address ✅
    SecondaryIpAddress Secondary IP Address IP Address ✅
    RedirectUrl Redirect URL Text ✅ Default: http://localhost:8080/

    🟢 MOFL

    OMS Name: MOFL
    Self Login: ❌ No

    Required Inputs

    Field Name Label Type Required Notes
    ClientId Client ID Text ✅ Minimum 3 characters
    Password Password Password ✅
    ApiKey API Key Text ✅
    PrimaryIpAddress Primary IP Address IP Address ✅
    SecondaryIpAddress Secondary IP Address IP Address ✅
    DOB Date of Birth Date ✅ Format: DD-MM-YYYY

    🟢 XTS

    OMS Name: XTS
    Self Login: ✅ Yes

    Required Inputs

    Field Name Label Type Required Notes
    UserId User ID Text ✅ Minimum 3 characters
    BaseURL Base URL Text ✅
    ApiKey API Key Text ✅
    ApiSecret API Secret Text ✅
    TotpSecret TOTP Secret Text ❌ Optional
    Source Source Text ✅
    InteractivePath Interactive Path Text ✅

    Notes

    • Self Login = Yes means the OMS supports direct authentication.
    • Self Login = No means credentials are validated via an external or broker-managed flow.
    • All fields marked Required must be provided during OMS configuration.
